# GridGarrison Agent Instructions

You are an expert code assistant for **GridGarrison**, a Spring Boot Modulith application that manages EV charging stations with blockchain-secured firmware verification and anomaly detection via Digital Twins.

## 📋 Project Overview

- **Architecture**: Spring Boot 3.3 with Spring Modulith 1.2.0 (strict module isolation)
- **Modules**: `orchestrator` (OCPP/WebSocket), `trust` (blockchain), `watchdog` (digital twin), `shared` (DTOs)
- **Communication Pattern**: Application Events (no cross-module direct imports)
- **Domain**: OCPP 2.0.1 EV charging protocol, Ethereum smart contracts, mTLS security

---

## ✅ Code Generation Principles

### 1. **Module Boundary Isolation (CRITICAL)**

The project enforces zero cross-module coupling using Spring Modulith. Services in different modules **must not** directly call each other.

#### ✅ **DO: Use Application Events**
```java
// In Orchestrator module → publish event
@Service
class OcppWebSocketHandler {
    @Autowired ApplicationEventPublisher eventPublisher;
    
    public void handleStationBoot(String stationId, String payload) {
        eventPublisher.publishEvent(new StationBootEvent(stationId, payload));
    }
}

// In Watchdog module → listen for event
@Service
class DigitalTwinServiceImpl implements DigitalTwinService {
    @EventListener
    void onStationBoot(StationBootEvent event) {
        registerStation(event.getStationId());
    }
}
```

#### ❌ **DON'T: Direct Service Imports Across Modules**
```java
// WRONG: Violates module boundary
@Service
class OcppHandler {
    @Autowired DigitalTwinService digitalTwinService;  // Cross-module import!
    
    public void boot(String id) {
        digitalTwinService.registerStation(id);  // Direct call = coupling
    }
}
```

**Action**: When you write code that imports services across modules, generate the correct event-based version and suggest refactoring the direct call.

---

### 2. **Service Pattern: Interface + Package-Private Implementation**

All services follow a public interface / package-private implementation pattern to enforce module contracts.

#### ✅ **Correct Pattern**
```java
// File: orchestrator/service/OcppService.java
package com.cybersecuals.gridgarrison.orchestrator.service;

public interface OcppService {
    /**
     * Public API of the {@code orchestrator} module.
     * Handles OCPP 2.0.1 frame validation and routing.
     */
    CompletableFuture<OcppResponse> routeMessage(OcppMessage message);
}

// File: orchestrator/service/OcppServiceImpl.java
package com.cybersecuals.gridgarrison.orchestrator.service;

@Service
class OcppServiceImpl implements OcppService {  // package-private
    @Override
    public CompletableFuture<OcppResponse> routeMessage(OcppMessage message) {
        // implementation
    }
}
```

**Key Rules**:
- Interface is `public`, lives in `service/` subpackage
- Implementation is `class` (not `public`), suffixed with `Impl`, same package
- Only annotate the implementation class with `@Service`, `@Autowired`, etc.
- Document the public interface with `@javadoc`

---

### 3. **Async Operations with CompletableFuture**

Services that perform I/O (blockchain calls, database queries, WebSocket operations) must return `CompletableFuture<T>`.

#### ✅ **Correct Pattern**
```java
public interface BlockchainService {
    /**
     * Verifies firmware hash against blockchain registry.
     * Non-blocking, returns immediately.
     */
    CompletableFuture<FirmwareHash> verifyGoldenHash(FirmwareHash hash);
}

@Service
class BlockchainServiceImpl implements BlockchainService {
    @Async
    @Override
    public CompletableFuture<FirmwareHash> verifyGoldenHash(FirmwareHash hash) {
        return CompletableFuture.supplyAsync(() -> {
            // Blockchain RPC call (blocking)
            String golden = getGoldenHashFromChain(hash.stationId());
            return hash.withGoldenValue(golden);
        });
    }
}
```

**Call Sites**:
```java
blockchainService.verifyGoldenHash(hash)
    .thenApply(verified -> { ... })
    .exceptionally(ex -> { ... });
```

**Rules**:
- Mark async methods with `@Async` on the implementation
- Always return `CompletableFuture<T>`, never `Future<T>`
- Use `CompletableFuture.supplyAsync()` to run blocking operations
- Chain with `.thenApply()`, `.thenCompose()`, `.exceptionally()`

### 4. **DTO / Domain Model Pattern**

All shared domain objects use **Lombok annotations** for immutability and construction clarity.

#### ✅ **Correct Pattern**
```java
// File: shared/dto/ChargingSession.java
package com.cybersecuals.gridgarrison.shared.dto;

@Value
@Builder
public class ChargingSession {
    UUID sessionId;
    String stationId;
    SessionState state;
    Instant startTime;
    
    /**
     * Session lifecycle states.
     */
    public enum SessionState { PENDING, ACTIVE, COMPLETED, FAULTED }
}
```

**Call Sites**:
```java
ChargingSession session = ChargingSession.builder()
    .sessionId(UUID.randomUUID())
    .stationId("GG-001")
    .state(ChargingSession.SessionState.PENDING)
    .startTime(Instant.now())
    .build();
```

**Rules**:
- Use `@Value` for immutability (final fields)
- Use `@Builder` for fluent construction
- Prefer nested `enum` for state/status fields
- Place only in `shared/dto/` package (cross-module visible)
- Add `@javadoc` to enum values

### 5. **Naming Conventions**

| Element | Pattern | Example | File Location |
|---------|---------|---------|---|
| **Interface** | Noun (Impl-agnostic capability) | `BlockchainService`, `DigitalTwinService` | `service/` |
| **Implementation** | `{Interface}Impl` | `BlockchainServiceImpl` | `service/` (package-private) |
| **Event Class** | `{Action}Event` (extends `ApplicationEvent`) | `StationBootEvent`, `AnomalyDetectedEvent` | `event/` |
| **DTO/Entity** | PascalCase, domain-specific | `ChargingSession`, `FirmwareHash` | `shared/dto/` |
| **Enum** | PascalCase type, UPPER_SNAKE values | `SessionState.PENDING` | Nested in class |
| **Method** | camelCase, imperative verb | `registerStation()`, `verifyGoldenHash()` | All |
| **Package** | `com.cybersecuals.gridgarrison.{module}.{layer}` | `com.cybersecuals.gridgarrison.trust.service` | All |

### 6. **Visibility Rules**

```
┌─────────────────────────────────────────────────────────────┐
│ VISIBILITY SCOPE                                            │
├─────────────────────────────────────────────────────────────┤
│ public      → Exported contract (interfaces, DTOs, configs) │
│ package     → Internal to module (Impl, handlers, helpers)  │
│ protected   → Never use (violates module boundaries)        │
│ private     → Within class only                             │
└─────────────────────────────────────────────────────────────┘
```

- Configuration classes annotated with `@Configuration`: `public`
- Service interfaces: `public`
- Service implementations: package-private (default access)
- DTOs in `shared/dto/`: `public`
- Internal handlers, helpers, contracts: package-private

## 🔍 Domain Concepts to Embed

When writing code, incorporate these domain concepts naturally:

### **OCPP 2.0.1 (EV Charging Protocol)**
- Frame format: `[messageTypeId, messageId, action, payload]`
- Key actions: `BootNotification`, `Heartbeat`, `TransactionEvent`, `FirmwareStatusNotification`
- Expect JSON-RPC over WebSocket (RFC 6455)

### **mTLS (Mutual TLS)**
- Station identity extracted from client certificate CN field
- Enforced by Spring Security x.509 configuration
- Example CN: `GG-STATION-001`

### **Digital Twin (Behavioral Model)**
- In-memory object tracking each station's live session
- Maintains energy baseline (kWh over time)
- Detects anomalies: energy spikes, clock drift, firmware mismatch
- Lightweight (not persisted to DB by default)

### **Blockchain (Ethereum Smart Contract)**
- Two-way operations via Web3j and RPC endpoint
- Read: `getGoldenHash(stationId)` → returns firmware hash + version
- Write: `registerGoldenHash(stationId, hash, version)` → stores new golden hash
- Used for firmware integrity verification

## 🎯 Code Generation Rules

### **For New Services**

1. **Create interface** in `{module}/service/` with full `@javadoc`
2. **Create package-private impl** with `@Service` annotation
3. **For I/O ops**: Return `CompletableFuture<T>`, mark with `@Async`
4. **For events**: Use `ApplicationEventPublisher.publishEvent()`
5. **For cross-module interaction**: Emit event; never inject service

### **For New Event Classes**

```java
package com.cybersecuals.gridgarrison.orchestrator.event;

public class StationBootEvent extends ApplicationEvent {
    private final String stationId;
    private final String payload;
    
    public StationBootEvent(Object source, String stationId, String payload) {
        super(source);
        this.stationId = stationId;
        this.payload = payload;
    }
    
    public String getStationId() { return stationId; }
    public String getPayload() { return payload; }
}
```

```java

@Value
@Builder
public class {Entity} {

    public enum Status { /* values */ }
}
```

## 🚨 Common Violations & Corrections

| Violation | Example | Fix |
|-----------|---------|-----|
| Direct service import | `@Autowired WatchdogService` in Orchestrator | Emit `ApplicationEvent`, listen in Watchdog |
| Synchronous I/O in main thread | `blockchainService.verify()` (no `.CompletableFuture`) | Return `CompletableFuture<T>` with `@Async` |
| Public Impl class | `public class BlockchainServiceImpl` | Remove `public`, make package-private |
| Missing @Builder on DTO | Manual constructor for `ChargingSession` | Add `@Builder` and `@Value` to DTO |
| Direct database call in WebSocket handler | Query DB inside `OcppWebSocketHandler` | Inject service, call async method, chain `.thenApply()` |

**When generating code**, if these violations appear, fulfill the request and immediately suggest the corrected pattern.

## 📚 Example: Correct Multi-Module Flow

**Scenario**: Orchestrator detects a station firmware mismatch, Trust module verifies it, Watchdog logs anomaly.

```java
// 1. Orchestrator: Parse OCPP frame, emit event
@Component
class OcppWebSocketHandler {
    @Autowired ApplicationEventPublisher publisher;
    
    public void handle(OcppMessage msg) {
        // ... parse ...
        publisher.publishEvent(new FirmwareStatusEvent(stationId, oldHash, newHash));
    }
}

// 2. Trust: Listen to event, verify hash asynchronously
@Service
class BlockchainServiceImpl implements BlockchainService {
    @EventListener
    void onFirmwareStatus(FirmwareStatusEvent event) {
        verifyGoldenHash(FirmwareHash.of(event.getNewHash()))
            .thenApply(verified -> 
                publisher.publishEvent(new FirmwareVerifiedEvent(...)))
            .exceptionally(ex -> {
                publisher.publishEvent(new FirmwareVerificationFailedEvent(...));
                return null;
            });
    }
}

// 3. Watchdog: Listen to verification result, update Digital Twin
@Service
class DigitalTwinServiceImpl implements DigitalTwinService {
    @EventListener
    void onFirmwareVerificationFailed(FirmwareVerificationFailedEvent event) {
        StationTwin twin = twins.get(event.getStationId());
        twin.recordAnomaly(AnomalyType.FIRMWARE_MISMATCH);
    }
}
```

**Result**: Zero cross-module coupling, fully asynchronous, auditablevia events.

## 🎓 When Generating Code

- **Always include `@javadoc`** on public interfaces and public methods
- **Prefer composition** over inheritance
